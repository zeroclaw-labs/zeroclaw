ZeroClaw es un asistente personal de IA que ejecutas en tus propios dispositivos. Te responde en los canales que ya usas (WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Matrix, IRC, Email, Bluesky, Nostr, Mattermost, Nextcloud Talk, DingTalk, Lark, QQ, Reddit, LinkedIn, Twitter, MQTT, WeChat Work y mГЎs). Tiene un panel web para control en tiempo real y puede conectarse a perifГ©ricos de hardware (ESP32, STM32, Arduino, Raspberry Pi). El Gateway es solo el plano de control вҖ” el producto es el asistente.

Si quieres un asistente personal, de un solo usuario, que se sienta local, rГЎpido y siempre activo, esto es lo que buscas.

## InstalaciГіn (recomendada)

Requisito: toolchain estable de Rust. Un solo binario, sin dependencias de runtime.

### Homebrew (macOS/Linuxbrew)

```bash
brew install zeroclaw
```

### Bootstrap con un clic

```bash
git clone https://github.com/zeroclaw-labs/zeroclaw.git
cd zeroclaw
./install.sh
```

`zeroclaw onboard` se ejecuta automГЎticamente despuГ©s de la instalaciГіn para configurar tu workspace y proveedor.

## Inicio rГЎpido (TL;DR)

GuГӯa completa para principiantes (autenticaciГіn, emparejamiento, canales): [Primeros pasos](docs/setup-guides/one-click-bootstrap.md)

```bash
# Instalar + onboard
./install.sh --api-key "sk-..." --provider openrouter

# Iniciar el gateway (servidor webhook + panel web)
zeroclaw gateway                # por defecto: 127.0.0.1:42617
zeroclaw gateway --port 0       # puerto aleatorio (seguridad reforzada)

# Hablar con el asistente
zeroclaw agent -m "Hello, ZeroClaw!"

# Modo interactivo
zeroclaw agent

# Iniciar runtime autГіnomo completo (gateway + canales + cron + hands)
zeroclaw daemon

# Verificar estado
zeroclaw status

# Ejecutar diagnГіsticos
zeroclaw doctor
```

ВҝActualizando? Ejecuta `zeroclaw doctor` despuГ©s de actualizar.

## MigraciГіn desde OpenClaw

ZeroClaw puede importar tu workspace, memoria y configuraciГіn de OpenClaw:

```bash
# Vista previa de lo que se migrarГЎ (seguro, solo lectura)
zeroclaw migrate openclaw --dry-run

# Ejecutar la migraciГіn
zeroclaw migrate openclaw
```

Esto migra tus entradas de memoria, archivos del workspace y configuraciГіn de `~/.openclaw/` a `~/.zeroclaw/`. La configuraciГіn se convierte de JSON a TOML automГЎticamente.

## Valores predeterminados de seguridad (acceso por DM)

ZeroClaw se conecta a superficies de mensajerГӯa reales. Trata los DMs entrantes como entrada no confiable.

GuГӯa completa de seguridad: [SECURITY.md](SECURITY.md)

Comportamiento predeterminado en todos los canales:

- **Emparejamiento por DM** (predeterminado): los remitentes desconocidos reciben un cГіdigo de emparejamiento corto y el bot no procesa su mensaje.
- Aprobar con: `zeroclaw pairing approve <channel> <code>` (luego el remitente se agrega a una lista de permitidos local).
- Los DMs pГәblicos entrantes requieren una activaciГіn explГӯcita en `config.toml`.
- Ejecuta `zeroclaw doctor` para detectar polГӯticas de DM riesgosas o mal configuradas.

**Niveles de autonomГӯa:**

| Nivel | Comportamiento |
|-------|----------------|
| `ReadOnly` | El agente puede observar pero no actuar |
| `Supervised` (predeterminado) | El agente actГәa con aprobaciГіn para operaciones de riesgo medio/alto |
| `Full` | El agente actГәa autГіnomamente dentro de los lГӯmites de la polГӯtica |

**Capas de sandboxing:** aislamiento del workspace, bloqueo de traversal de rutas, listas de comandos permitidos, rutas prohibidas (`/etc`, `/root`, `~/.ssh`), limitaciГіn de velocidad (mГЎximo de acciones/hora, topes de costo/dГӯa).

## CaracterГӯsticas destacadas

- **Runtime ligero por defecto** вҖ” los flujos de trabajo comunes de CLI y estado se ejecutan en una envolvente de memoria de pocos megabytes en compilaciones release.
- **Despliegue econГіmico** вҖ” diseГұado para placas de $10 e instancias pequeГұas en la nube, sin dependencias de runtime pesadas.
- **Arranque en frГӯo rГЎpido** вҖ” el runtime de Rust con un solo binario mantiene el inicio de comandos y del daemon casi instantГЎneo.
- **Arquitectura portable** вҖ” un binario para ARM, x86 y RISC-V con proveedores/canales/herramientas intercambiables.
- **Gateway local-first** вҖ” un solo plano de control para sesiones, canales, herramientas, cron, SOPs y eventos.
- **Bandeja de entrada multicanal** вҖ” WhatsApp, Telegram, Slack, Discord, Signal, iMessage, Matrix, IRC, Email, Bluesky, Nostr, Mattermost, Nextcloud Talk, DingTalk, Lark, QQ, Reddit, LinkedIn, Twitter, MQTT, WeChat Work, WebSocket y mГЎs.
- **OrquestaciГіn multi-agente (Hands)** вҖ” enjambres de agentes autГіnomos que se ejecutan segГәn programaciГіn y se vuelven mГЎs inteligentes con el tiempo.
- **Procedimientos Operativos EstГЎndar (SOPs)** вҖ” automatizaciГіn de flujos de trabajo dirigida por eventos con MQTT, webhook, cron y disparadores de perifГ©ricos.
- **Panel web** вҖ” interfaz web React 19 + Vite con chat en tiempo real, explorador de memoria, editor de configuraciГіn, gestor de cron e inspector de herramientas.
- **PerifГ©ricos de hardware** вҖ” ESP32, STM32 Nucleo, Arduino, Raspberry Pi GPIO a travГ©s del trait `Peripheral`.
- **Herramientas de primera clase** вҖ” shell, E/S de archivos, navegador, git, web fetch/search, MCP, Jira, Notion, Google Workspace y mГЎs de 70 mГЎs.
- **Hooks de ciclo de vida** вҖ” intercepta y modifica llamadas LLM, ejecuciones de herramientas y mensajes en cada etapa.
- **Plataforma de skills** вҖ” skills incluidos, comunitarios y del workspace con auditorГӯa de seguridad.
- **Soporte de tГәneles** вҖ” Cloudflare, Tailscale, ngrok, OpenVPN y tГәneles personalizados para acceso remoto.

### Por quГ© los equipos eligen ZeroClaw

- **Ligero por defecto:** binario pequeГұo de Rust, arranque rГЎpido, bajo consumo de memoria.
- **Seguro por diseГұo:** emparejamiento, sandboxing estricto, listas de permitidos explГӯcitas, alcance del workspace.
- **Totalmente intercambiable:** los sistemas centrales son traits (proveedores, canales, herramientas, memoria, tГәneles).
- **Sin dependencia de proveedor:** soporte de proveedores compatibles con OpenAI + endpoints personalizados conectables.

## AutenticaciГіn por suscripciГіn (OpenAI Codex / Claude Code / Gemini)

ZeroClaw soporta perfiles de autenticaciГіn nativos de suscripciГіn (multi-cuenta, cifrados en reposo).

- Archivo de almacenamiento: `~/.zeroclaw/auth-profiles.json`
- Clave de cifrado: `~/.zeroclaw/.secret_key`
- Formato de id de perfil: `<provider>:<profile_name>` (ejemplo: `openai-codex:work`)

```bash
# OpenAI Codex OAuth (suscripciГіn ChatGPT)
zeroclaw auth login --provider openai-codex --device-code

# Gemini OAuth
zeroclaw auth login --provider gemini --profile default

# Anthropic setup-token
zeroclaw auth paste-token --provider anthropic --profile default --auth-kind authorization

# Verificar / refrescar / cambiar perfil
zeroclaw auth status
zeroclaw auth refresh --provider openai-codex --profile default
zeroclaw auth use --provider openai-codex --profile work

# Ejecutar el agente con autenticaciГіn por suscripciГіn
zeroclaw agent --provider openai-codex -m "hello"
zeroclaw agent --provider anthropic -m "hello"
```

    ```powershell

    ```

    DespuГ©s de la instalaciГіn, abre una nueva terminal y ejecuta `rustup default stable` para asegurarte de que el toolchain estable estГ© activo.

3. **Verifica** que ambos funcionen:
    ```powershell
    rustc --version
    cargo --version
    ```

#### Opcional

- **Docker Desktop** вҖ” requerido solo si usas el [runtime sandbox con Docker](#soporte-de-runtime-actual) (`runtime.kind = "docker"`). Instala vГӯa `winget install Docker.DockerDesktop`.

</details>

<details>
<summary><strong>Linux / macOS</strong></summary>

#### Requerido

1. **Herramientas de compilaciГіn esenciales:**
    - **Linux (Debian/Ubuntu):** `sudo apt install build-essential pkg-config`
    - **Linux (Fedora/RHEL):** `sudo dnf group install development-tools && sudo dnf install pkg-config`
    - **macOS:** Instala Xcode Command Line Tools: `xcode-select --install`

2. **Toolchain de Rust:**

    ```bash
    curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
    ```

    Ver [rustup.rs](https://rustup.rs) para detalles.

3. **Verifica** que ambos funcionen:
    ```bash
    rustc --version
    cargo --version
    ```

#### Instalador en una lГӯnea

O salta los pasos anteriores e instala todo (dependencias del sistema, Rust, ZeroClaw) en un solo comando:

```bash

```

#### Requisitos de recursos para compilaciГіn

Compilar desde el cГіdigo fuente necesita mГЎs recursos que ejecutar el binario resultante:

| Recurso        | MГӯnimo  | Recomendado |
| -------------- | ------- | ----------- |
| **RAM + swap** | 2 GB    | 4 GB+       |
| **Disco libre**| 6 GB    | 10 GB+      |

Si tu host estГЎ por debajo del mГӯnimo, usa binarios preconstruidos:

```bash

```

Para requerir instalaciГіn solo de binarios sin compilaciГіn de respaldo:

```bash

```

#### Opcional

- **Docker** вҖ” requerido solo si usas el [runtime sandbox con Docker](#soporte-de-runtime-actual) (`runtime.kind = "docker"`). Instala vГӯa tu gestor de paquetes o [docker.com](https://docs.docker.com/engine/install/).

> **Nota:** El `cargo build --release` predeterminado usa `codegen-units=1` para reducir la presiГіn mГЎxima de compilaciГіn. Para compilaciones mГЎs rГЎpidas en mГЎquinas potentes, usa `cargo build --profile release-fast`.
