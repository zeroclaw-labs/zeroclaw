# Phase 1.2 — Summary

**Completed:** 2026-06-12
**Status:** ✅ Quick-gate 19/19. All 3 M1 requirements (WS-01, WS-04, WS-05) covered.

## Critical inventory finding

Direct grep of the merged v0.7.5 source for `inventory!`, `linkme::distributed_slice`, and `ctor::ctor` returned **ZERO** matches. Upstream uses explicit registration today. The pitfall #6 risk flagged in PITFALLS.md (auto-discovery defeating structural exclusion) DOES NOT APPLY to v0.7.5. Phase 1.2 collapses from "migrate to Pattern 1" to "ratify the existing pattern in deny.toml and CI." This saves substantial work in Phase 1.3+ — the structural MCP boundary is achievable purely through Cargo.toml dependency-edge management.

### Workspace registration (WS-01)
- `Cargo.toml` `[workspace].members` extended with `"bins/engineer"` and `"bins/wizard"` (preserves all existing upstream members; Phase 1.4 strips the dead ones).
- `resolver = "2"` already pinned by upstream — no change needed.

### Distributed-slice ban ratification (WS-04)
- `deny.toml` `[bans].deny` extended with `inventory`, `linkme`, `ctor` (each with explicit reason). Any PR that introduces these crates fails `cargo deny check bans`.
- `.github/workflows/osagent-policy.yml` new job `no-distributed-slice-registration`: greps the entire workspace source tree for `inventory::submit!`, `linkme::distributed_slice`, `#[ctor::ctor]` invocations. Catches even hand-rolled or copy-pasted instances.

### Workspace-build gate
- `.github/workflows/osagent-policy.yml` new job `workspace-build`: runs `cargo build -p osagent-engineer --release && cargo build -p osagent-wizard --release`. Cheap (<30s at Phase 1.2 hollow state), grows with the project.

### Default-features audit prep (WS-05)
- Engineer and wizard `Cargo.toml` `[dependencies]` blocks are empty at Phase 1.2 (hollow). The convention is documented in both files' header comments: every entry uses `default-features = false` and lists features explicitly. Active enforcement (`cargo tree --duplicates`) ships in Phase 1.5 alongside MANIFEST.toml.

### Harness extensions
- `scripts/validate-quick.sh` extended with 8 new Phase 1.2 checks: bins/ existence, package names, wizard-no-MCP dep, deny.toml ban list, source-level inventory!/linkme!/ctor! grep, workspace.members entry.
