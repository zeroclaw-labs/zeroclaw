## What's red on `osagent-main` and why

`cargo build --workspace` will fail until the cascading source-level references to dropped channels/providers/tools are cleaned. Specifically:

1. `crates/zeroclaw-runtime/src/tools/mod.rs` still has `pub use zeroclaw_tools::<dropped_tool>::*` lines for several deleted tools.
2. Various source files use `#[cfg(feature = "channel-discord")]` etc. for features that no longer exist.
3. Tool/channel/provider registration call sites in `zeroclaw-runtime` still reference deleted symbols.

The `wizard-no-mcp-gate` (Phase 1.3, milestone-defining) remains green throughout — wizard binary is hollow and has no source-level deps on any dropped crate. The bash test suite (5 suites / 298 assertions) is green.
