## What stayed in `[workspace.metadata.osagent]` vs SemVer build metadata

Per CONTEXT.md the original recommendation was `version = "0.1.0+zeroclaw-0.7.5"` using SemVer build-metadata syntax. The planner and executor chose bare `version = "0.1.0"` with provenance in a structured `[workspace.metadata.osagent]` block instead. Rationale: SemVer build metadata is ignored for precedence ordering AND is stripped by most registry tooling — the provenance would be invisible to consumers. The metadata block survives upstream merges and is grep-able from any tool reading `Cargo.toml`. Plan-checker flagged this divergence as `[info]` non-blocking.
