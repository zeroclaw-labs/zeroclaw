# Phase 1.1 — Validation Strategy

> Per-phase validation contract for feedback sampling during execution.

This is a **pure-infrastructure phase** — no Rust source code, no runtime tests. Validation is shell-script verification of repository state, CI configuration, and documentation existence. Drawn from RESEARCH.md `## Validation Architecture` section.

---

## Test Infrastructure

| Property | Value |
|----------|-------|
| **Framework** | bash + `gh` CLI + `cargo deny` + `jq` |
| **Config file** | `scripts/validate-quick.sh` and `scripts/validate-full.sh` (created in Wave 0) |
| **Quick run command** | `bash scripts/validate-quick.sh` |
| **Full suite command** | `bash scripts/validate-full.sh` |
| **Estimated runtime** | ~30s quick / ~90s full (network-bound on `gh api` + `cargo deny check`) |

---

## Sampling Rate

- **After every task commit:** Run `bash scripts/validate-quick.sh` (5–10s — checks file existence + git state)
- **After every plan wave:** Run `bash scripts/validate-full.sh` (60–90s — full cargo-deny check + gh API verification)
- **Before `/gsd:verify-work`:** Full suite must be green
- **Max feedback latency:** 90s
