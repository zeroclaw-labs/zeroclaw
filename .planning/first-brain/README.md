# First Brain — 나 자신을 위한 LLM 위키

> "내가 누구인지"를 시간에 따라 함께 정리하고 연결하는 개인 지식 베이스.
> 문서는 사람이 큐레이션하고, 편집·연결·유지보수는 LLM이 담당한다.

## 1. 이 위키가 다루는 주제

외부 지식이 아니라 **"나 자신"** 이다. 다음을 축적하고 연결한다.

- **정체성(Identity)** — 나는 누구인가, 어떤 가치·성향·역사를 가졌는가
- **관계(Relationships)** — 가족, 친구, 동료, 중요한 타인
- **경험(Experiences)** — 사건, 여행, 이정표, 전환점
- **작업(Work)** — 프로젝트, 직업적 시도, 결과물, 학습
- **생각(Thoughts)** — 아이디어, 질문, 의견, 반성
- **흐름(Themes)** — 반복되는 패턴, 관심사의 궤적

## 2. 구조 (3계층)

```
.planning/first-brain/
├── AGENTS.md              # LLM 스키마 (위키를 어떻게 관리할지)
├── README.md              # 이 문서
├── raw/                   # 원본 자료 (불변, 사람이 넣는다)
│   ├── journal/           # 일기, 메모, 짧은 기록
│   ├── artifacts/         # 사진, 문서, 기사 클리핑
│   └── conversations/     # 대화 로그, 회의록
└── wiki/                  # LLM이 관리하는 마크다운 (가변)
    ├── index.md           # 전체 페이지 카탈로그
    ├── log.md             # 시간순 이벤트 로그
    ├── overview.md        # "나"의 최상위 요약
    ├── people/            # 인물 페이지 (나, 가족, 친구, ...)
    ├── experiences/       # 사건·경험 페이지
    ├── work/              # 프로젝트·작업 페이지
    ├── thoughts/          # 생각·아이디어 페이지
    ├── themes/            # 반복 패턴·관심사 흐름
    └── sources/           # 각 원본 자료에 대한 요약 페이지
```

- **raw/** 는 읽기 전용이다. LLM이 수정하지 않는다.
- **wiki/** 는 LLM이 쓰고 유지한다. 사람이 읽고 질문한다.
- **AGENTS.md** 는 두 계층 사이의 규약이다.

## 3. 기본 워크플로우

### (1) Ingest — 새 자료를 넣을 때
1. `raw/` 아래 알맞은 폴더에 자료를 떨어뜨린다 (예: `raw/journal/2026-04-18.md`).
2. LLM에게 "이거 정리해줘"라고 말한다.
3. LLM은 원본을 읽고 → 핵심을 나와 토론하고 → `wiki/sources/`에 요약 페이지를 만들고 → 관련된 인물/경험/작업/생각 페이지를 갱신하고 → `index.md`와 `log.md`에 항목을 추가한다.

### (2) Query — 질문할 때
- "나는 3년간 어떤 사람이 되어가고 있지?" 같은 질문을 던진다.
- LLM은 `index.md`를 먼저 읽고, 관련 페이지를 타고 들어가 답한다.
- **좋은 탐색 결과는 위키에 새 페이지로 저장한다.** (채팅으로 증발하지 않게)

### (3) Lint — 주기적 건강 검진
- "모순된 주장 있어? 오래된 내용은? 고아 페이지는?"
- LLM은 위키를 훑고 정리 제안을 한다.

## 4. 프라이버시 원칙

이 시스템은 매우 민감한 개인 데이터를 다룬다. `AGENTS.md`의 프라이버시 규약을 반드시 먼저 읽는다.

- 실명/연락처/계정정보/금융정보는 `raw/`에 직접 넣지 않는 것을 권장(암호화 또는 로컬 전용).
- 공개 커밋/푸시 전에 민감 정보를 스캔한다.
- 위키는 원칙적으로 **비공개 저장소에서만** 유지한다.
- 이 저장소가 공개/공유되는 경우 `.gitignore`에 `.planning/first-brain/raw/` 를 추가할 것을 고려한다.

## 5. 시작하는 법

1. `wiki/people/me.md` 를 열어 기초 프로필을 채운다(LLM과 대화하며).
2. 오늘 일기를 `raw/journal/YYYY-MM-DD.md`로 하나 쓰고 LLM에게 Ingest를 요청한다.
3. 가족/친구 중 한 명의 페이지를 만들며 워크플로를 몸에 익힌다.
4. 2주쯤 돌려보고 `AGENTS.md`를 자신의 스타일에 맞게 수정한다.

이 위키는 사람과 LLM이 함께 **공진화**한다. 스키마도, 구조도, 태그 체계도 시간에 따라 쓰임새에 맞게 바뀌어야 한다.
