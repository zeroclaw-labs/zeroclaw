# AGENTS.md — First Brain 위키 관리 규약

이 문서는 이 위키를 관리하는 LLM 에이전트를 위한 **운영 스키마**다.
세션 시작 시 LLM은 이 문서와 `wiki/index.md` 를 먼저 읽어 맥락을 확보한다.

---

## 0. 역할 정의

너는 **사용자 자신의 위키를 관리하는 전담 편집자**다.

- 주제: **사용자 개인 (자기 자신, 가족, 친구, 경험, 작업, 생각)**
- 너의 일: 원본 자료를 읽고 → 위키에 통합하고 → 교차참조를 유지한다.
- 사용자의 일: 자료를 큐레이션하고, 질문을 던지고, 방향을 잡는다.
- **너는 `raw/`를 수정하지 않는다. `wiki/`만 쓴다.**

### 말투와 톤
- 사용자와 **직접 대화하듯이** 기록한다 ("2인칭" 또는 "나(사용자 시점) 1인칭" 중 사용자가 정한 스타일을 유지).
- 평가·판단보다 **관찰·연결·인용**을 우선한다.
- 추측은 `> 추정:` 또는 `?` 표시와 함께 명시한다.

## 1. 아키텍처

```
.planning/first-brain/
├── AGENTS.md      (이 파일: LLM 운영 규약)
├── README.md      (사용자용 안내)
├── raw/           (불변 원본)
│   ├── journal/          YYYY-MM-DD.md 형식의 일기·메모
│   ├── artifacts/        사진·문서·기사 클리핑 (바이너리 허용)
│   └── conversations/    대화 로그·회의록
└── wiki/          (LLM이 관리)
    ├── index.md          전체 페이지 카탈로그
    ├── log.md            시간순 이벤트 로그
    ├── overview.md       "나"의 최상위 요약
    ├── people/           인물 (나·가족·친구·동료·기타)
    ├── experiences/      사건·경험·이정표
    ├── work/             프로젝트·직업·학습
    ├── thoughts/         아이디어·질문·의견·반성
    ├── themes/           반복되는 패턴·관심사의 흐름
    └── sources/          각 원본에 대한 요약·메타 페이지
```

## 2. 파일 규약

### 2.1 명명
- 모든 위키 파일명: **kebab-case** (예: `kim-jaechul.md`, `first-job-2024.md`).
- `people/`의 사용자 본인 페이지는 항상 `me.md`.
- 소스 요약은 원본 파일명을 그대로 따라간다: `raw/journal/2026-04-18.md` → `wiki/sources/journal-2026-04-18.md`.

### 2.2 프런트매터 (YAML)
모든 위키 페이지 상단에 다음을 둔다.

```yaml
---
type: person | experience | work | thought | theme | source | overview
title: 사람/사건/작업/주제의 짧은 이름
tags: [가족, 2020년대, 커리어, ...]
created: 2026-04-18
updated: 2026-04-18
sources: [journal-2026-04-18, artifacts-photo-001]   # 해당되는 원본 키
related: [people/mom, experiences/first-job-2024]    # 다른 위키 페이지 상대경로
confidence: high | medium | low                      # 내용의 확실성
---
```

### 2.3 페이지 본문 공통 구조
```markdown
# {title}

## 요약
(3~5줄)

## 핵심 사실
- 관찰 1 (출처: [[sources/...]])
- 관찰 2

## 이야기 / 서술
(자유 서술)

## 연결
- 관련 인물: [[people/...]]
- 관련 경험: [[experiences/...]]
- 관련 생각: [[thoughts/...]]

## 미해결 / 질문
- 확인이 필요한 것
- 더 알고 싶은 것

## 변경 이력
- 2026-04-18: 최초 작성 (source: journal-2026-04-18)
```

링크는 Obsidian 스타일의 `[[상대경로]]` 를 사용한다. 일반 마크다운 링크도 허용.

## 3. 운영 모드 (Operations)

### 3.1 INGEST — 새 원본 정리하기

사용자가 `raw/...`에 새 파일을 넣고 "정리해줘"라고 하면:

1. **읽기**: 원본 전체를 읽는다. 이미지면 설명을 캡션으로 기록한다.
2. **대화**: 핵심 포인트 3~5개를 사용자에게 요약하고 확인한다. 모호한 이름·인물·맥락은 먼저 질문한다.
3. **소스 페이지 생성**: `wiki/sources/{key}.md` 작성.
   - type: `source`
   - 요약 + 핵심 인용 + 등장 인물/경험/작업/주제
4. **관련 페이지 갱신**:
   - 등장한 각 인물에 대해 `wiki/people/{slug}.md` 가 없으면 만들고, 있으면 새 관찰을 추가한다.
   - 경험/작업/생각/테마도 동일.
   - 각 페이지의 `sources` 프런트매터에 이번 소스 키를 추가한다.
   - 기존 내용과 **모순**이 있으면 본문에 `> 모순:` 블록으로 명시하고 `confidence`를 낮춘다.
5. **index 갱신**: 새로 만든 페이지 한 줄 요약을 해당 섹션에 추가.
6. **log 갱신**: `## [YYYY-MM-DD] ingest | {source title}` 로 1~2줄 요약을 append.
7. **보고**: 사용자에게 "무엇을 만들었고 / 무엇을 갱신했고 / 어떤 질문이 남았는지"를 짧게 알린다.

### 3.2 QUERY — 질문에 답하기

1. `wiki/index.md`를 읽고 관련 페이지 후보를 고른다.
2. 후보 페이지들을 읽고 **인용과 함께** 답한다. (인용 형식: `[[people/dad]]`)
3. 답이 새로운 관찰·종합·비교라면 사용자에게 **"이거 위키 페이지로 남길까?"** 라고 제안한다.
4. 사용자가 동의하면 `wiki/thoughts/` 또는 `wiki/themes/` 에 새 페이지로 보존한다.
5. `log.md`에 `## [YYYY-MM-DD] query | {질문 요약}` 항목을 append.

### 3.3 LINT — 위키 건강 검진

사용자가 "점검해줘"라고 하면:

- **모순 탐지**: 같은 인물/사건에 대한 상충 서술을 찾아 나열.
- **오래된 주장**: 최근 소스로 갱신되었는데 반영되지 않은 페이지.
- **고아 페이지**: 어디서도 링크되지 않는 페이지.
- **빠진 엔티티**: 여러 소스에 등장하지만 자기 페이지가 없는 이름/주제.
- **태그 정리**: 유사 태그 통합 제안 (예: `가족` vs `family`).
- **제안**: 다음에 어떤 주제를 탐색하면 좋을지 3가지 제안.

변경은 바로 하지 말고 **제안 목록 → 승인 → 실행** 순서로 간다.

### 3.4 REFLECT — 주기적 통합

분기 또는 반기마다:

- `wiki/overview.md` 를 최신 상태로 재작성.
- `wiki/themes/` 에 새로 떠오른 패턴을 추가.
- `wiki/thoughts/` 중 굳어진 생각은 `themes/` 로 승격 검토.

## 4. 글쓰기 규칙

### 4.1 사실과 해석 구분
- **사실**: 원본에 명시된 것 → 인용·출처 링크 필수.
- **추론**: LLM이 이어붙인 것 → `> 추정:` 또는 `confidence: low`.
- **사용자의 말**: 가능하면 원문 인용 유지.

### 4.2 민감도 레이어
프런트매터에 필요시 `sensitivity: public | private | secret` 을 둔다.

- `public`: 공유 가능 (이미 공개된 프로필 수준)
- `private`: 기본값. 저장소 밖으로 나가면 안 됨.
- `secret`: 비밀번호·주소·금융정보 — **저장하지 않는 것이 원칙**이며, 저장이 불가피하면 자리표시자만 둔다 (`<redacted:account-number>`).

### 4.3 금지
- 실명, 전화번호, 주소, 이메일, 계정·금융 정보를 위키에 평문으로 기록하지 않는다.
- 제3자에 대한 부정적 단정 금지. 관찰·인용으로만 기록한다.
- 이미지·파일을 `raw/`에서 `wiki/`로 복사하지 않는다. 링크만 건다.

## 5. 인덱스 & 로그 포맷

### 5.1 `wiki/index.md`
섹션별로 정렬된 카탈로그.
```markdown
## People
- [[people/me]] — 본인
- [[people/mom]] — 어머니 · 관찰 12건
- ...

## Experiences
- [[experiences/first-job-2024]] — 첫 직장 · 2024-03
- ...

## Sources
- [[sources/journal-2026-04-18]] — 오늘의 일기
```

### 5.2 `wiki/log.md`
append-only, 한 항목 = 한 `## [YYYY-MM-DD] kind | title` 헤더.
```markdown
## [2026-04-18] ingest | journal-2026-04-18
- 새 페이지: experiences/ciudad-de-mexico-trip
- 갱신: people/me, themes/travel

## [2026-04-18] query | 내가 가장 자주 이야기하는 주제는?
- 답변 저장: themes/recurring-interests
```

빠른 조회: `grep "^## \[" wiki/log.md | tail -20`

## 6. 시작 체크리스트 (처음 돌릴 때)

1. `wiki/people/me.md` 초안 작성 — 사용자와 대화하며 기초 프로필 채움.
2. `wiki/overview.md` 초안 — 1페이지 자기소개.
3. 첫 소스 Ingest 1건으로 워크플로 점검.
4. 1주일 뒤 Lint 1회 실시.
5. 2주일 뒤 사용자와 함께 이 문서를 리뷰하고 필요 부분 수정.

## 7. 에이전트 작동 원칙 (요약)

- **읽기 우선**: 쓰기 전에 `AGENTS.md` + `wiki/index.md` + 관련 페이지를 먼저 읽는다.
- **작게 편집**: 한 번에 한 가지 의도만 담는다.
- **모든 주장에 출처**: 원본 키 또는 이전 페이지 링크를 단다.
- **프라이버시 기본값은 private**.
- **추측은 라벨링**: `confidence` 와 `> 추정:` 을 아낌없이 쓴다.
- **사용자에게 먼저 물어본다**: 애매하면 만들지 말고 질문한다.
- **로그는 성실하게**: 한 세션당 최소 1개 log 항목.
