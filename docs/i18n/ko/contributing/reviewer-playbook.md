# 리뷰어 플레이북

이 플레이북은 [`pr-workflow.md`](./pr-workflow.md)의 운영 동반 문서입니다.
더 넓은 문서 탐색은 [`docs/README.md`](../README.md)를 사용합니다.

## 0. 요약

- **목적:** 대량 PR에서 리뷰 품질을 높게 유지하는 결정적 리뷰어 운영 모델을 정의합니다.
- **대상:** 메인테이너, 리뷰어, 에이전트 보조 리뷰어.
- **범위:** 접수 분류, 리스크-깊이 라우팅, 심층 리뷰 검사, 자동화 오버라이드, 핸드오프 프로토콜.
- **비목표:** `CONTRIBUTING.md`의 PR 정책 권한이나 CI 파일의 워크플로우 권한을 대체하는 것.

---

## 1. 리뷰 상황별 빠른 경로

전체 상세 내용을 읽기 전에 빠르게 라우팅하려면 이 섹션을 사용합니다.

### 1.1 처음 5분 이내에 접수 실패

1. 하나의 실행 가능한 체크리스트 코멘트를 남깁니다.
2. 접수 차단 사항이 수정될 때까지 심층 리뷰를 중단합니다.

이동:

- [섹션 3.1](#31-5분-접수-분류)

### 1.2 리스크가 높거나 불명확한 경우

1. 기본적으로 `risk: high`로 취급합니다.
2. 심층 리뷰와 명시적인 롤백 증거를 요구합니다.

이동:

- [섹션 2](#2-리뷰-깊이-결정-매트릭스)
- [섹션 3.3](#33-심층-리뷰-체크리스트-고위험)

### 1.3 자동화 출력이 잘못되었거나 노이즈가 있는 경우

1. 오버라이드 프로토콜을 적용합니다 (`risk: manual`, 코멘트/라벨 중복 제거).
2. 명시적인 근거와 함께 리뷰를 계속합니다.

이동:

- [섹션 5](#5-자동화-오버라이드-프로토콜)

### 1.4 리뷰 핸드오프가 필요한 경우

1. 범위/리스크/검증/차단 사항과 함께 핸드오프합니다.
2. 구체적인 다음 조치를 할당합니다.

이동:

- [섹션 6](#6-핸드오프-프로토콜)

## 2. 리뷰 깊이 결정 매트릭스

| 리스크 라벨 | 일반적으로 변경되는 경로 | 최소 리뷰 깊이 | 필수 증거 |
|---|---|---|---|
| `risk: low` | 문서/테스트/잡무, 격리된 비런타임 변경 | 리뷰어 1명 + CI 게이트 | 일관된 로컬 검증 + 동작 모호성 없음 |
| `risk: medium` | `src/providers/**`, `src/channels/**`, `src/memory/**`, `src/config/**` | 서브시스템 인지 리뷰어 1명 + 동작 검증 | 집중 시나리오 증명 + 명시적 부작용 |
| `risk: high` | `src/security/**`, `src/runtime/**`, `src/gateway/**`, `src/tools/**`, `.github/workflows/**` | 빠른 분류 + 심층 리뷰 + 롤백 준비 | 보안/장애 모드 검사 + 롤백 명확성 |

확실하지 않은 경우 `risk: high`로 취급합니다.

자동 리스크 라벨링이 컨텍스트상 잘못된 경우, 메인테이너가 `risk: manual`을 적용하고 최종 `risk:*` 라벨을 명시적으로 설정할 수 있습니다.

## 3. 표준 리뷰 워크플로우

### 3.1 5분 접수 분류

모든 새 PR에 대해:

1. 템플릿 완성도를 확인합니다 (`summary`, `validation`, `security`, `rollback`).
2. 라벨이 존재하고 타당한지 확인합니다:
   - `size:*`, `risk:*`
   - 범위 라벨 (예: `provider`, `channel`, `security`)
   - 모듈 범위 라벨 (`channel:*`, `provider:*`, `tool:*`)
   - 해당하는 경우 기여자 등급 라벨
3. CI 신호 상태를 확인합니다 (`CI Required Gate`).
4. 범위가 하나의 관심사인지 확인합니다 (정당성이 없는 한 혼합 대형 PR은 거부).
5. 개인정보/데이터 위생 및 중립적 테스트 문구 요건이 충족되는지 확인합니다.

접수 요건이 실패하면, 심층 리뷰 대신 하나의 실행 가능한 체크리스트 코멘트를 남깁니다.

### 3.2 빠른 레인 체크리스트 (모든 PR)

- 범위 경계가 명시적이고 신뢰할 수 있습니다.
- 검증 명령이 존재하고 결과가 일관됩니다.
- 사용자 대면 동작 변경이 문서화되었습니다.
- 작성자가 동작과 영향 범위에 대한 이해를 보여줍니다 (특히 에이전트 보조 PR의 경우).
- 롤백 경로가 구체적입니다 (단순히 "revert"가 아닌).
- 호환성/마이그레이션 영향이 명확합니다.
- diff 아티팩트에 개인/민감 데이터 유출이 없습니다; 예제/테스트가 중립적이고 프로젝트 범위입니다.
- 정체성 관련 문구가 존재하는 경우, ZeroClaw/프로젝트 고유 역할을 사용합니다 (개인적이거나 실제 정체성이 아닌).
- 명명 및 아키텍처 경계가 프로젝트 계약을 따릅니다 (`AGENTS.md`, `CONTRIBUTING.md`).

### 3.3 심층 리뷰 체크리스트 (고위험)

고위험 PR의 경우, 각 카테고리에서 최소 하나의 구체적인 예시를 검증합니다:

- **보안 경계:** 기본 거부(deny-by-default) 동작이 보존되었고, 의도하지 않은 범위 확장이 없습니다.
- **장애 모드:** 오류 처리가 명시적이며 안전하게 성능이 저하됩니다.
- **계약 안정성:** CLI/config/API 호환성이 보존되었거나 마이그레이션이 문서화되었습니다.
- **관측성:** 비밀 유출 없이 장애를 진단할 수 있습니다.
- **롤백 안전성:** revert 경로와 영향 범위가 명확합니다.

### 3.4 리뷰 코멘트 결과 스타일

하나의 명시적 결과와 함께 체크리스트 스타일 코멘트를 선호합니다:

- **merge 준비 완료** (이유를 설명합니다).
- **작성자 조치 필요** (순서가 있는 차단 사항 목록).
- **더 깊은 보안/런타임 리뷰 필요** (정확한 리스크와 요청된 증거를 명시합니다).

불필요한 왕복 지연을 만드는 모호한 코멘트를 피합니다.

## 4. 이슈 분류 및 백로그 거버넌스

### 4.1 이슈 분류 라벨 플레이북

백로그를 실행 가능하게 유지하기 위해 라벨을 사용합니다:

- `r:needs-repro`는 불완전한 버그 리포트용.
- `r:support`는 버그 백로그 외부로 라우팅하는 것이 나은 사용법/지원 질문용.
- `duplicate` / `invalid`는 실행 불가능한 중복/노이즈용.
- `no-stale`은 외부 차단 사항을 기다리는 수락된 작업용.
- 로그/페이로드에 개인 식별자 또는 민감 데이터가 포함된 경우 수정을 요청합니다.

### 4.2 PR 백로그 정리 프로토콜

리뷰 수요가 용량을 초과하는 경우, 다음 순서를 적용합니다:

1. 활성 버그/보안 PR(`size: XS/S`)을 큐 상단에 유지합니다.
2. 겹치는 PR에 통합을 요청하고, 확인 후 오래된 것을 `superseded`로 닫습니다.
3. stale 닫기 기간이 시작되기 전에 휴면 PR에 `stale-candidate`를 표시합니다.
4. stale/대체된 기술 작업을 다시 열기 전에 rebase + 새로운 검증을 요구합니다.

---

## 5. 자동화 오버라이드 프로토콜

자동화 출력이 리뷰 부작용을 만드는 경우 사용합니다:

1. **잘못된 리스크 라벨:** `risk: manual`을 추가한 다음 의도한 `risk:*` 라벨을 설정합니다.
2. **이슈 분류에서 잘못된 자동 닫기:** 이슈를 다시 열고, 라우트 라벨을 제거하고, 하나의 명확한 코멘트를 남깁니다.
3. **라벨 스팸/노이즈:** 하나의 정식 메인테이너 코멘트를 유지하고 중복 라우트 라벨을 제거합니다.
4. **모호한 PR 범위:** 심층 리뷰 전에 분할을 요청합니다.

---

## 6. 핸드오프 프로토콜

리뷰를 다른 메인테이너/에이전트에게 넘기는 경우, 다음을 포함합니다:

1. 범위 요약.
2. 현재 리스크 클래스와 근거.
3. 이미 검증된 사항.
4. 열린 차단 사항.
5. 제안된 다음 조치.

---

## 7. 주간 큐 위생

- stale 큐를 검토하고 수락되었지만 차단된 작업에만 `no-stale`을 적용합니다.
- `size: XS/S` 버그/보안 PR을 먼저 우선순위로 지정합니다.
- 반복되는 지원 이슈를 문서 업데이트 및 자동 응답 안내로 전환합니다.
