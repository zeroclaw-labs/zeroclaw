# 安全性公告維護者檢查清單

在處理從私密通報到公開揭露的高影響漏洞時，請使用此檢查清單。

## A) 收案與分級（私密階段）

- [ ] 確認通報位於私密公告工作流程中（非公開 issue）。
- [ ] 驗證可重現性與影響程度。
- [ ] 分類嚴重性等級與受影響的元件。
- [ ] 記錄嚴重性判定依據與 SLA 計時起始時間。
- [ ] 決定處理路徑：
  - [ ] 接受並建立草稿公告
  - [ ] 要求補充細節
  - [ ] 以非安全性問題結案並附上理由
- [ ] 在公告討論中新增初始時間表/負責人。

## B) 禁止公開期間的修補開發

- [ ] 啟動或使用公告專用的暫時私密分支。
- [ ] 確保利用細節不出現在公開 PR/issue 討論中。
- [ ] 實作最小風險的修補程式與測試。
- [ ] 在本機執行必要的驗證：
  - [ ] `cargo test --workspace --all-targets`
  - [ ] `cargo test -- security`
  - [ ] `cargo test -- tools::shell`
  - [ ] `cargo test -- tools::file_read`
  - [ ] `cargo test -- tools::file_write`
- [ ] 如有支援版本需要，準備向後移植的修補。

## C) 公告中繼資料品質

- [ ] 使用公告中繼資料範本：`docs/security/advisory-metadata-template.md`。
- [ ] 受影響的套件/生態系欄位正確無誤。
- [ ] 受影響的版本範圍精確無誤。
- [ ] 已填入修復版本，或已記錄緩解措施。
- [ ] 已盡可能填寫 CWE/CVSS 欄位。
- [ ] 參考資料中包含修補 commit 與版本說明。

## D) 揭露與後續追蹤

- [ ] 在修復/緩解措施就緒時發布公告。
- [ ] 在適當時申請 CVE（或附上既有的 CVE）。
- [ ] 確認已發布的公告引用了已釋出的修復產出物。
- [ ] 確認下游通知/相依性訊號已對齊。
- [ ] 監控回歸問題或繞過通報，如範圍變更則更新公告中繼資料。

## E) 內部衛生管理

- [ ] commit、日誌、CI 輸出或討論串中不含機密資訊。
- [ ] 揭露前不在公開頻道中洩漏不必要的利用細節。
- [ ] 安全回應時間軸與決策日誌已記錄在公告留言中。
