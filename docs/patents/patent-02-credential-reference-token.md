# MoA 특허명세서 — 제2호

## 참조 토큰 기반 자격증명 보호형 AI 에이전트 브라우저 자동화 방법 및 시스템

### 기술 분야

본 발명은 AI 에이전트가 사용자를 대신하여 웹사이트 로그인, 결제, 문서 다운로드 등의 작업을 수행할 때, 실제 자격증명(비밀번호, 카드번호, 주민등록번호 등)이 외부 LLM에 노출되지 않도록 하는 참조 토큰 기반 보안 아키텍처에 관한 것이다.

### 배경 기술

AI 에이전트가 사용자를 대신하여 웹사이트에 로그인하거나 결제를 수행하려면, 필연적으로 비밀번호와 카드번호 등의 민감 정보를 처리해야 한다. 기존 방식에서는:

1. **LLM 직접 전달 방식**: 사용자의 비밀번호를 LLM에게 전달하고, LLM이 브라우저 도구에 비밀번호를 인자로 전달. 이 경우 비밀번호가 외부 LLM 서버의 대화 기록에 평문으로 남게 되어 심각한 보안 위험이 존재.
2. **토큰화 없는 암호화 방식**: 로컬에 암호화하여 저장하되, 사용 시 복호화된 값을 LLM에게 전달하여 브라우저에 입력. 복호화 시점에서 LLM에 노출되는 문제는 동일.

두 방식 모두 LLM이라는 외부 시스템에 민감 정보가 전달되는 근본적 문제를 해결하지 못한다.

### 해결하고자 하는 과제

본 발명은 AI 에이전트가 로그인, 결제 등을 수행할 때, **LLM은 참조 토큰만 취급**하고 **실제 자격증명의 복호화와 브라우저 폼 입력은 로컬 게이트웨이 내부에서만** 이루어지도록 하여, 민감 정보가 외부 LLM에 절대 노출되지 않는 보안 아키텍처를 제공하는 것을 과제로 한다.

### 과제 해결 수단

#### 1. 암호화 자격증명 금고 (Encrypted Credential Vault)

사용자의 민감 정보(ID, 비밀번호, 카드번호, 주민등록번호, 여권번호 등)를 ChaCha20-Poly1305 인증 암호화 방식으로 로컬 기기에만 저장한다. 저장 시:
- 사이트 도메인(site)과 라벨(label)을 키로 사용
- 평문은 즉시 암호화되어 저장
- 마스킹된 표시 힌트(예: `****-1234`)만 별도 보관

#### 2. 참조 토큰 발급

LLM이 자격증명을 요청하면, 실제 값 대신 **참조 토큰**을 반환한다:

```
요청: credential_recall get site=coupang.com label=password
응답: {{CRED:coupang.com:password}}  (참조 토큰)
```

LLM은 이 참조 토큰을 브라우저 도구의 인자로 전달한다:

```
browser fill @e2 {{CRED:coupang.com:password}}
```

#### 3. 로컬 게이트웨이 토큰 해석

AI 에이전트의 로컬 게이트웨이 프로세스가 브라우저 도구 실행 직전에 참조 토큰을 가로채어:
1. `{{CRED:site:label}}` 패턴을 감지
2. 로컬 암호화 금고에서 해당 항목을 복호화
3. 복호화된 값을 브라우저 프로세스(Playwright/Chromium)에 직접 전달
4. 브라우저가 웹 폼에 값을 입력
5. 복호화된 값은 메모리에서 즉시 폐기 (변수 스코프 종료)

#### 4. 동의 기반 사용 프로토콜

저장된 자격증명이 있더라도 사용 전 반드시 사용자 동의를 요구한다:
- 어떤 사이트에 어떤 행동을 할 것인지 명시
- 어떤 자격증명을 사용할 것인지 마스킹 표시로 확인
- 결제 시: 금액별 단계적 확인 (소액: 단순 확인, 10만원 이상: 이중 확인)

#### 5. 외부 전송 금지 규칙

- 자격증명은 로컬 암호화 금고에만 저장
- 외부 동기화 서버(Railway 릴레이)에 전송 금지
- LLM 대화 기록(memory_store, memory_observe)에 저장 금지
- LLM에 전송되는 대화 기록에는 참조 토큰만 존재하며, 설령 LLM이 기억하더라도 실제 값이 아닌 토큰만 보유

### 발명의 효과

1. **LLM 무지 원칙**: LLM은 실제 자격증명을 절대 알 수 없음 (참조 토큰만 취급)
2. **온라인 쇼핑/결제 자동화**: 실물 쇼핑 결제까지 AI가 대행 가능
3. **CAPTCHA/2FA 처리**: 스크린샷으로 사용자에게 보여주고 응답을 받아 입력
4. **다층 보안**: 암호화 저장 + 참조 토큰 + 동의 기반 사용 + 외부 전송 금지
5. **정부 발행 ID 보호**: 주민등록번호, 여권번호 등도 동일한 보안 체계로 보호

### 청구항

1. AI 에이전트가 사용자를 대신하여 웹사이트 인증 및 거래를 수행하는 방법에 있어서,
   (a) 사용자의 자격증명을 인증 암호화 방식으로 로컬 기기의 암호화 금고에 저장하는 단계;
   (b) AI 언어 모델이 자격증명을 요청할 때, 실제 값 대신 참조 토큰({{CRED:site:label}} 형식)을 반환하는 단계;
   (c) AI 언어 모델이 상기 참조 토큰을 브라우저 자동화 도구의 인자로 전달하는 단계;
   (d) 로컬 게이트웨이가 브라우저 도구 실행 직전에 참조 토큰을 감지하여, 로컬에서 복호화하고, 복호화된 값을 브라우저 프로세스에 직접 전달하는 단계;
   (e) 복호화된 값을 브라우저 폼 입력 후 즉시 메모리에서 폐기하는 단계;
   를 포함하며, 상기 모든 단계에서 실제 자격증명이 외부 AI 언어 모델에 전달되지 않는 것을 특징으로 하는 방법.

2. 제1항에 있어서, 상기 자격증명을 사용하기 전에, 사용자에게 수행할 행동, 대상 사이트, 사용할 자격증명(마스킹 표시)을 명시하고 명시적 동의를 받는 단계를 더 포함하는 것을 특징으로 하는 방법.

3. 제2항에 있어서, 금전 거래 시 거래 금액에 따라 단계적 확인을 요구하되, 소정 금액(예: 10만원) 이상의 거래에 대해서는 이중 확인을 요구하는 것을 특징으로 하는 방법.

4. 제1항에 있어서, 상기 자격증명이 비밀번호, 신용카드번호, CVC, 주민등록번호, 운전면허증번호, 여권번호 중 하나 이상을 포함하며, 이들 모두에 대해 동일한 참조 토큰 방식이 적용되는 것을 특징으로 하는 방법.

5. 제1항에 있어서, 상기 자격증명이 외부 동기화 서버, AI 언어 모델의 대화 기록, 외부 메모리 저장소에 전송되는 것이 금지되며, LLM에 전송되는 대화 기록에는 참조 토큰만 포함되는 것을 특징으로 하는 방법.

6. 제1항에 있어서, 웹사이트의 CAPTCHA 또는 2단계 인증 요구 시, 브라우저 스크린샷을 사용자에게 제시하고, 사용자로부터 응답을 수신하여 브라우저에 입력하는 단계를 더 포함하는 것을 특징으로 하는 방법.

7. AI 에이전트가 참조 토큰을 통해 자격증명을 취급하며, 로컬 게이트웨이가 토큰을 실시간 복호화하여 브라우저에 전달하는 프로세서, 암호화 금고, 및 브라우저 자동화 모듈을 포함하는 자격증명 보호형 AI 에이전트 시스템.

## 도면의 간단한 설명

### [도 1] 참조 토큰 기반 자격증명 흐름도

```
사용자 ─── "비밀번호: mypass123" ───→ MoA 에이전트
                                        │
                                        ▼
                              credential_store로 암호화 저장
                              (ChaCha20-Poly1305 → enc2:...)
                                        │
                                        ▼
LLM (외부) ←── {{CRED:site:password}} ──┘  (참조 토큰만 전달)
    │
    │ browser fill @e2 {{CRED:site:password}}
    ▼
로컬 게이트웨이 ── 토큰 감지 → 로컬 복호화 ──→ Chromium 폼 입력
                                                    │
                                          복호화 값 즉시 폐기
```

### [도 2] 동의 기반 사용 프로토콜

```
저장된 자격증명 발견
    │
    ▼
사용자에게 확인:
"쿠팡 계정 user@email.com으로 로그인할까요?"
    │
    ├── 사용자 동의 → 진행
    │
    └── 사용자 거부 → 중지
         │
         ▼
결제 시 (₩100,000 이상):
"₩150,000 결제입니다. '결제 확인'이라고 입력해주세요."
    │
    ├── "결제 확인" 입력 → 결제 진행
    │
    └── 미입력/거부 → 결제 중지
```
