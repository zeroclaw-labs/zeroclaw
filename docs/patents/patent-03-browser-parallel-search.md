# MoA 특허명세서 — 제3호

## 영속 브라우저 데몬 기반 병렬 다중 엔진 웹 검색 및 단계적 심층 탐색 방법과 시스템

### 기술 분야

본 발명은 AI 에이전트가 사용자의 질문에 답변하기 위해 웹 검색을 수행할 때, 영속적(persistent) 브라우저 데몬을 통해 다수의 검색 엔진에서 동시에 검색하고, 검색 결과를 단계적으로 심층 탐색하며, 수직 깊이와 수평 페이지네이션을 구분하여 관리하는 방법 및 시스템에 관한 것이다.

### 배경 기술

기존의 AI 에이전트 웹 검색 방식에는 다음과 같은 한계가 있다:

1. **API 기반 검색의 한계**: DuckDuckGo, Brave 등의 검색 API는 봇 탐지에 의해 차단되거나, 유료 API 키가 필요하며, 검색 결과만 반환하고 상세 페이지 내용은 별도의 스크래핑 단계가 필요.
2. **순차적 처리**: 검색 → 결과 수신 → 스크래핑 → 스크래핑 결과 수신의 2단계 순차 처리로 4초 이상 소요.
3. **단일 검색 엔진 의존**: 하나의 검색 엔진에 의존하므로, 해당 엔진의 특성(예: DuckDuckGo의 한국어 검색 취약)이나 장애에 취약.
4. **단순 깊이 제한**: 검색 결과 링크를 일률적으로 N개까지만 방문하는 방식으로, 수평 탐색(페이지네이션)과 수직 탐색(링크 추적)을 구분하지 않음.

### 해결하고자 하는 과제

본 발명은 영속적 브라우저 데몬을 활용하여 (1) 다수의 검색 엔진에서 병렬로 동시 검색하고, (2) AI가 검색 결과에서 최적의 링크를 선별하여 단계적으로 심층 탐색하며, (3) 수직 깊이와 수평 페이지네이션을 독립적으로 관리하여 사용자의 의사결정에 따라 탐색 범위를 조절하는 방법을 제공하는 것을 과제로 한다.

### 과제 해결 수단

#### 1. 영속 브라우저 데몬 아키텍처

브라우저 프로세스(Chromium)를 에이전트 시작 시 또는 첫 검색 요청 시 한 번만 실행하고, HTTP 서버 형태의 데몬으로 유지한다:

```
AI 에이전트 루프 (Rust) → HTTP POST → Playwright 데몬 (Node.js) ↔ Chromium (CDP)
```

- 데몬은 소정 포트(예: 9500)에서 HTTP 요청을 수신
- 명령(navigate, text, click, screenshot 등)을 Chromium에 전달
- 쿠키, 로그인 세션, 접근성 트리 상태를 영속 유지
- 유휴 시간(예: 30분) 경과 시 자동 종료

#### 2. 3-Phase 웹 리서치 워크플로우

**Phase 1 — 쿼리 계획**: 사용자의 질문에서 암묵적 컨텍스트(위치, 시간, 인물)를 장기 메모리에서 조회하여 최적화된 검색어를 생성.

**Phase 2 — 병렬 다중 엔진 검색**: 브라우저 데몬에 복수의 탭을 동시에 열어, 각 탭에서 서로 다른 검색 엔진(예: 네이버, 구글, 덕덕고)의 검색 결과 페이지에 동시 접속. 모든 탭의 페이지 로딩이 완료되면 각 페이지의 텍스트를 추출하여 병합.

단일 사이트 검색과 동일한 시간(~2초) 안에 3배의 검색 결과를 획득하며, 각 검색 엔진의 강점(예: 네이버의 한국어 최적화, 구글의 영어/범용 검색)을 동시에 활용.

**Phase 3 — 스마트 딥다이브**: AI가 병합된 검색 결과에서 사용자 질문에 가장 관련성이 높은 상위 5개 링크를 선별하고, 해당 링크를 방문하여 상세 내용을 추출. 필요 시 추가 링크를 추적하되, 수직 깊이와 수평 페이지네이션을 독립적으로 관리.

#### 3. 수직-수평 분리 탐색 관리

**수직 탐색 (깊이 제한: 3단계)**
- Level 1: 검색 결과 페이지에서 상위 5개 링크 방문
- Level 2: 상세 페이지에서 "더보기", "관련 링크" 등 추적
- Level 3: 2차 참조 링크 1단계 추가 추적
- 3단계 초과 시 자동 중지 (토큰 보존)

**수평 탐색 (사용자 확인 기반)**
- 번호형 페이지네이션: 10페이지 자동 탐색 후 사용자에게 계속 여부 확인
- 무한 스크롤: 10회 스크롤 후 사용자에게 계속 여부 확인
- 사용자 승인 시 다음 10페이지/10회 스크롤 반복

수평 탐색을 수직 깊이 제한에서 분리함으로써, 페이지네이션이 있는 검색 결과를 충분히 탐색하면서도 불필요한 링크 추적을 방지.

#### 4. 접근성 트리 기반 @Ref 시스템

브라우저의 접근성 트리(Accessibility Tree)에서 상호작용 가능한 요소(버튼, 링크, 입력 필드 등)에 안정적인 참조 번호(@e1, @e2 등)를 할당. CSS 선택자와 달리:
- Content Security Policy에 의한 차단 없음
- React/Vue 하이드레이션에 의한 손실 없음
- Shadow DOM 내부 요소 접근 가능
- DOM 구조 변경에 강건 (역할 기반)

@Ref의 유효성은 사용 전 비동기 count() 검사로 확인하며, 페이지 네비게이션 시 자동 초기화.

### 발명의 효과

1. **봇 차단 회피**: 실제 Chromium 브라우저 사용으로 검색 엔진의 봇 탐지에 의한 차단 없음
2. **병렬 검색**: 3개 검색 엔진 동시 검색으로 단일 엔진 대비 3배의 결과를 동일 시간에 획득
3. **비용 절감**: 모든 검색이 무료 (유료 API 키 불필요)
4. **한 단계 완성**: 검색 + 스크래핑이 한 번의 브라우저 접속으로 완료 (기존 2단계 → 1단계)
5. **지능적 탐색**: AI가 검색 결과를 선별하여 관련성 높은 링크만 방문
6. **사용자 통제**: 수평 탐색의 범위를 사용자가 실시간으로 결정
7. **세션 유지**: 로그인 상태, 쿠키가 영속 유지되어 인증 사이트 검색도 가능

### 청구항

1. AI 에이전트가 웹 검색을 수행하는 방법에 있어서,
   (a) 영속적 브라우저 데몬을 통해 복수의 브라우저 탭을 동시에 열어, 각 탭에서 서로 다른 검색 엔진의 검색 결과 페이지에 동시 접속하는 병렬 검색 단계;
   (b) 모든 탭의 페이지 텍스트를 추출하여 하나의 통합 검색 결과로 병합하는 단계;
   (c) AI 언어 모델이 통합 검색 결과에서 사용자 질문에 대한 관련성을 기준으로 소정 수(예: 5개)의 최적 링크를 선별하는 단계;
   (d) 선별된 링크를 방문하여 상세 내용을 추출하되, 수직 깊이(링크 추적)는 소정 단계(예: 3단계)로 제한하고, 수평 탐색(페이지네이션/스크롤)은 소정 단위(예: 10페이지)마다 사용자 확인을 받는 단계;
   를 포함하는 것을 특징으로 하는 방법.

2. 제1항에 있어서, 상기 검색 수행 전에 사용자의 질문에서 암묵적 컨텍스트(위치, 시간, 인물)를 장기 메모리에서 조회하여 검색어를 최적화하는 쿼리 계획 단계를 더 포함하는 것을 특징으로 하는 방법.

3. 제1항에 있어서, 상기 복수의 검색 엔진이 네이버(한국어 최적화), 구글(영어/범용), 덕덕고(프라이버시 보호)를 포함하며, 각 검색 엔진의 검색 URL에 최적화된 쿼리 파라미터를 설정하는 것을 특징으로 하는 방법.

4. 제1항에 있어서, 상기 수직 탐색의 각 단계에서 AI 언어 모델이 페이지 내용의 관련성을 판단하여 추가 링크 추적 여부를 결정하는 것을 특징으로 하는 방법.

5. 제1항에 있어서, 상기 브라우저 데몬이 접근성 트리(Accessibility Tree) 기반의 @Ref 시스템을 통해 웹 페이지의 상호작용 요소에 안정적인 참조 번호를 할당하고, CSS 선택자를 대체하여 사용하는 것을 특징으로 하는 방법.

6. 제1항에 있어서, 상기 브라우저 데몬이 최초 검색 요청 시 자동 시작되고, 유휴 상태 소정 시간(예: 30분) 경과 시 자동 종료되며, 크래시 발생 시 다음 요청에서 자동 재시작되는 것을 특징으로 하는 방법.

7. 영속적 브라우저 데몬, 다중 탭 병렬 검색 모듈, 수직-수평 분리 탐색 관리 모듈, 접근성 트리 기반 @Ref 시스템을 포함하는 AI 에이전트용 웹 검색 시스템.

## 도면의 간단한 설명

### [도 1] 병렬 3-사이트 검색 아키텍처

```
사용자: "서울 강남구 내일 날씨"
    │
    ▼
Phase 1: 쿼리 최적화
    │ memory_recall → 위치: 서울 강남구
    │ "내일" → 2026-03-30
    │ 최적 쿼리: "서울 강남구 2026년 3월 30일 날씨 예보"
    ▼
Phase 2: 병렬 브라우저 검색 (~2초)
    ┌──────────────┬──────────────┬──────────────┐
    │   Tab 1      │   Tab 2      │   Tab 3      │
    │   Naver      │   Google     │  DuckDuckGo  │
    │   검색 URL   │   검색 URL   │   검색 URL   │
    │   페이지 로드 │   페이지 로드 │   페이지 로드 │
    │   텍스트 추출 │   텍스트 추출 │   텍스트 추출 │
    └──────┬───────┴──────┬───────┴──────┬───────┘
           │              │              │
           └──────────────┼──────────────┘
                          ▼
                    결과 병합 → LLM
    ▼
Phase 3: 스마트 딥다이브
    상위 5개 링크 선별 → 방문 → 상세 내용 추출
    │
    └─ 수직 3단계 / 수평 10페이지씩 사용자 확인
```

### [도 2] 수직-수평 분리 탐색

```
수직 탐색 (3단계 자동):
  L1: 검색결과 ──→ 상세페이지
  L2: 상세페이지 ──→ 관련링크
  L3: 관련링크 ──→ 참조문서  ← STOP

수평 탐색 (10단위 사용자 확인):
  [1~10페이지] ──→ "계속?" ──→ [11~20페이지] ──→ "계속?" → ...
  [스크롤 1~10] ──→ "계속?" ──→ [스크롤 11~20] ──→ "계속?" → ...
```
