ZeroClaw / Multi-agent / Skill-bundles

Many agents, shared skill-bundles

One ZeroClaw install runs many named agents — [agents.<alias>]. Each is independently isolated; capability is composed from bundles. A skill-bundle is a named, reusable set of skills any number of agents can attach by alias.

Multi-agent — one install, many isolated agents [agents.<alias>]
channelsplatforms it answers on
model_providerits own model & provider
risk_profileautonomy & approval gates
memoryits own scoped store
workspacejailed filesystem
skill_bundleswhich skills it can use →
No global "default agent" — every agent is explicitly named, and channels route to whichever agent claims them. Two agents on one host can carry entirely different capabilities.
Skill-bundles[skill_bundles.<alias>]
  • A named, reusable skill set — defined once, attached by alias to any agent (or shared by many)
  • Lives at <install>/shared/skills/<alias>/ by default (override with directory, kept inside shared/)
  • include / exclude filter exactly which skills the bundle exposes from that dir
Wiring it upconfig.toml
[agents.support]
channels       = ["discord.general"]
risk_profile   = "support"
skill_bundles  = ["common", "support-only"]  # attach by alias

[agents.devops]
risk_profile   = "devops"
skill_bundles  = ["common", "devops-advanced"]

[skill_bundles.support-only]
include = ["search-docs", "escalate"]  # only these
Bundles are reusable — many-to-many
support→ commonsupport-only
devops→ commondevops-advanced
common is defined once and shared by both — one bundle, many agents.
Why it matters

Skills, model, memory, filesystem, and autonomy are scoped per agent — a low-trust support bot can't reach devops deploy skills, all from one install.
