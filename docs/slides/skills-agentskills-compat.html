Skills are agentskills.io-compatible

A skill is just a folder with a SKILL.md — flat frontmatter where only name and description are required, per the open Agent Skills spec. The same skill installs from the public ClawHub marketplace or the official zeroclaw‑labs/zeroclaw‑skills registry.

SKILL.md — the unit of portability flat frontmatter
---
name: release-check          # required
description: Gate a release  # required
version: 1.2.0              # optional
author: zeroclaw-labs       # optional
license: MIT                 # optional
tags: [Official, slash]      # tiering / surfaces
---
# Markdown instructions the agent reads…
One flat schema drives the CLI, dashboard form, and TUI. No YAML dependency — the hand-rolled parser covers every scalar field. Portable in, portable out.
ClawHub public marketplace clawhub.ai
$ zeroclaw skills install clawhub:web-search
  • Resolves slug → clawhub.ai/api/v1/download, pulls a ZIP (≤ 50 MiB)
  • Extracted with a path-traversal guard, then audited before install
  • Accepts clawhub:slug or a full clawhub.ai/owner/slug URL
zeroclaw‑skills registry official default github.com/zeroclaw‑labs
$ zeroclaw skills install release-check  # bare name
  • Clones/syncs the registry repo locally, refreshed daily
  • registry.json carries trust tiers — Official / Featured / Community
  • Tier drives the install banner; non-Official skills warn before they land
Source resolution — first match wins
1clawhub:› 2git URL› 3bare name → registry› 4registry:name/skill› 5local path
Bundle-aware

Installs land in a skill bundle via --bundle / the agent's assigned bundle — so a running agent actually loads them, not an orphaned global dir.
