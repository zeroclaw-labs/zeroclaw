## 1. 한 줄 요약

코드베이스는 **빌드 청결, 구조적으로 건강합니다.** 최근 vault(세컨드브레인) 작업은 sync 저널에 정확히 통합됐고, L1 zero-knowledge 릴레이는 와이어업 완료(2026-03-02 plan의 가장 큰 갭이 닫힘)되었으며, 대부분의 보안 프리미티브(페어링 토큰, 시크릿 저장소, AEAD 암호, 웹훅 서명 검증)는 검증 완료입니다.

다만 **빌드 직전에 반드시 처리해야 하는 12개의 critical 이슈**를 발견했습니다. 가장 중요한 두 가지:

- **🚨 멀티 디바이스 동기화가 설계 누락으로 동작하지 않음 (D2):** 각 디바이스가 자기 `.sync_key`를 random 생성하는데 **페어링 흐름에 키 분배 단계가 없음.** 테스트 코드 주석(`src/memory/synced.rs:784`)이 직접 시인합니다 — 테스트는 같은 키를 시뮬레이트하지만 production에는 디바이스 간 키 공유 경로가 없습니다. 효과: 사용자의 핵심 기능("MoA를 여러 디바이스에 깔면 기억이 동기화된다")이 **현재 동작하지 않음**.
- **🚨 SLM Executor에 safe-for-slm 필터링이 없음 (E1+E2):** SLM Executor는 정의되어 있지만 현재 어디서도 호출되지 않아 실용적 위험은 0입니다. 그러나 Phase 3가 요구하는 와이어업이 진행되는 순간, SLM이 `shell` / `file_write` / `file_edit` / `apply_patch` / `delegate` / `cron_*` 같은 모든 위험 도구를 직접 호출할 수 있게 됩니다. **와이어업과 필터는 반드시 같은 PR에서 함께 처리되어야 합니다.**

그리고 동등하게 심각하지만 우리 코드가 아닌 **의존성 레이어의 critical 발견:**

- **🚨 27건의 알려진 보안 취약점 (V1+V2+V3, `cargo audit` 결과):** `aws-lc-sys`에 5건 (TLS 인증서 검증 우회, AES-CCM 타이밍 사이드채널, PKCS7 우회), `rustls-webpki 0.102.8`과 `0.103.9`에 각 4건 (인증서 제약 우회 + reachable panic), **`wasmtime 36.0.6`에 12건 (sandbox-escape 등급 포함)**. 우리 MoA 코드 자체의 결함은 아니지만, 모든 릴리스 바이너리에 그대로 실립니다. 자세한 표는 §6.5.

나머지 7개 critical 이슈는 우리 코드: sync를 침묵 비활성화시키는 config Default/serde 드리프트(C1), 경제 분류기의 `partial_cmp().unwrap()` panic 표면(F1), WhatsApp QR 페어링 인증값 평문 로그(F2), 특허 사양과 다른 sync 키 유도 방식(D1), 약한 패스워드 해시(D3), Android bridge의 빌드 타깃 컴파일 실패(G4).

11개의 medium 이슈는 대부분 보강(CSRF, 에러 메시지 누설, 다운그레이드 경로) + production-completion-plan.md가 이미 식별한 클라이언트 플랫폼 갭(Tauri WebSocket IPC, Android NDK 빌드, iOS bridge 확인).

**권고:** 9개 critical 이슈를 집중 PR 시리즈로 빌드 직전에 처리하세요. Medium / minor는 후속 PR로 충분합니다.
