# [Tracker]: ZeroClaw TUI

ZeroClaw needs a standalone terminal user interface that provides parity with the web dashboard and serves as the primary operator interface for power users, headless servers, and enclave deployments. The TUI is a separate binary (`zeroclaw-tui`) that communicates with the ZeroClaw daemon exclusively over a socket protocol — no in-process runtime embedding. All client surfaces (TUI, web dashboard, CLI) share common gateway API primitives — the gateway is the single source of truth. No client gets special endpoints; if the TUI needs it, the gateway exposes it, and the web dashboard and CLI can consume it too.

### Architecture

- [x] Shared widget/theme library remains in `crates/zeroclaw-tui` for reuse by the main binary's onboarding
- [ ] Transport abstraction: Unix socket (local), WebSocket/WSS (remote), vsock (enclave/future) — #6293
- [x] Exact version match enforcement on connect via `/api/status`
- [x] `--ephemeral` daemon mode (#6818): TUI spawns `zeroclaw daemon --ephemeral` if no daemon is running; daemon self-terminates (configurable grace period) when last TUI client disconnects; daemon started without `--ephemeral` ignores client count
- [ ] Multiple TUI instances per daemon
- [ ] Move `crates/zeroclaw-tui` to `apps/tui` as a separate `publish = false` binary crate (#6821)

### Connection Modes

- [ ] `zeroclaw-tui` — local Unix socket (default)
- [ ] `zeroclaw-tui --connect wss://host:port` — remote daemon
- [ ] `zeroclaw-tui --connect vsock://cid:port` — enclave (future, #6293)
- [ ] Authentication for remote connections (pairing token validation on WSS/remote connect — moved from #6837 socket transport, where filesystem permissions handle local auth)

### Connection Resilience

- [ ] Socket close reason propagation — both sides know intentional vs unintentional disconnect
- [ ] Intentional disconnect: show reason, no retry
- [ ] Unintentional disconnect: banner, retry with backoff for configurable `reconnect_timeout_secs`, composer disabled but history remains scrollable
- [ ] Reconnect without restarting TUI — re-establish socket, resume session by ID
- [ ] `--ephemeral` daemon crash: one automatic re-spawn attempt, then require manual intervention

### Core Requirements

- [ ] Session-scoped runtime overrides without daemon reload (#6817) — gateway-level feature shared by TUI, web dashboard, and CLI
- [x] Ratatui 0.30+ / crossterm 0.29+
- [ ] Syntax highlighting via syntect (code fences, diff viewer)
- [ ] Markdown parsing via pulldown-cmark (code fence detection)
- [ ] Code fence rendering must be copy-clean — no padding characters in selectable content; use background color banding for visual separation
- [ ] `$EDITOR` handoff (full-screen suspend/resume) for file editing; no embedded editor
- [ ] Internationalization via Fluent — same catalogs (en/fr/ja) as CLI, all TUI strings through Fluent from the start
- [ ] File/attachment upload protocol (#6819) — gateway-level feature; binary over socket, daemon persists to agent workspace
