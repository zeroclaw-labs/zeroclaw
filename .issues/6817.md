# [Feature]: Session-scoped runtime overrides without daemon reload

Per-session runtime parameter changes (model, temperature) without requiring a config edit or daemon reload. Overrides are session-scoped — they shadow config-level defaults for that session only, other sessions are unaffected, and overrides evaporate when the session ends.

### Implementation

**`SessionOverrides` struct** (`crates/zeroclaw-runtime/src/rpc/session.rs`):
```rust
pub struct SessionOverrides {
    pub model: Option<String>,
    pub temperature: Option<f64>,
}
```

**Stored on `RpcSession`** — each RPC session carries its own `overrides` field, defaulting to `Default::default()`.

**`session/configure` RPC method** — set/replace overrides for an active session. Returns the confirmed overrides in the response (`SessionConfigureResult`).

**`set_overrides()` / `get_overrides()`** on `SessionStore` — thread-safe access to per-session override state.
