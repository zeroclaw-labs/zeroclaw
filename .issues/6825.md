# [Tracker]: TUI UX

TUI UX — cross-cutting user experience concerns for the ZeroClaw terminal interface. Covers theming, keybindings, navigation, accessibility, and overall polish.

### Current state

- Theme system exists (`theme.rs`): icy-blue ZeroClaw palette with style helpers
- Widget library (`widgets.rs`, 83 lines): basic shared widgets
- Onboarding UI (`onboarding.rs`, 724 lines): first-run configuration wizard

### Scope

**Theming & visual polish**
- [ ] Dark/light theme support (detect terminal background)
- [ ] Configurable color scheme (user overrides in config)
- [ ] Consistent use of theme palette across all views (chat, onboarding, status)

**Navigation & layout**
- [ ] Session sidebar: list of active/recent sessions with switch-on-click
- [ ] Tab or panel system for multiple views (chat, config, logs, status)
- [ ] Responsive layout: adapt to terminal size changes

**Keybindings**
- [ ] Configurable keybindings
- [ ] Help overlay (?) showing available keybindings in context
- [ ] Vim-style navigation option

**Accessibility**
- [ ] Screen reader compatibility (semantic content ordering)
- [ ] High-contrast mode
- [ ] Mouse support (click, scroll)

**Status & feedback**
- [ ] Connection status indicator (connected/disconnected/reconnecting)
- [ ] Active model display in status bar
- [ ] Cost/token usage display
- [ ] Turn progress indicator (thinking, calling tools, generating)
