# [Feature]: --ephemeral daemon mode

A `zeroclaw daemon --ephemeral` flag. When set, the daemon tracks connected RPC clients and self-terminates (after a grace period) when the last client disconnects. Daemons started without `--ephemeral` ignore client count and run until explicitly stopped.

### Implementation

- `daemon/mod.rs`: `EPHEMERAL_GRACE_SECS = 30` — grace period before shutdown after last client disconnects
- `unix.rs`: `client_count: Arc<AtomicUsize>` incremented on connect, decremented on disconnect
- `DaemonSubsystems::socket_start` closure receives `client_count` for the daemon to monitor
- Daemon `select!` loop watches an `ephemeral_shutdown` future that resolves when `client_count` drops to 0 (after at least one connection), then sleeps the grace period before returning `DaemonExit::EphemeralShutdown`
- Socket file is cleaned up on shutdown via `std::fs::remove_file`
