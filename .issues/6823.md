# [Tracker]: TUI ACP Bridge

TUI ACP Bridge — the client-side connection layer that connects the TUI to the ZeroClaw daemon over the RPC Unix socket (or WSS for remote). Translates between TUI application events and JSON-RPC 2.0 protocol messages.

This is the client-side counterpart to #6837's server-side dispatch layer.

### Scope

- [ ] RPC client: connect to daemon Unix socket, send/receive NDJSON frames
- [ ] `initialize` handshake: version negotiation, receive server capabilities
- [ ] `session/new` + `session/prompt`: create sessions and send user messages
- [ ] Notification stream: receive `session/update` notifications (text chunks, tool calls, tool results, approval requests) and route them to the TUI rendering layer
- [ ] `session/cancel`: cancel in-flight turns
- [ ] `session/configure`: send runtime overrides (model, temperature)
- [ ] Tool approval flow: receive `ApprovalRequest` notifications, present to user, send approval/rejection back
- [ ] Connection lifecycle: detect disconnects, reconnect with session resumption
- [ ] Daemon auto-spawn: if no daemon is running, spawn `zeroclaw daemon --ephemeral` and connect (#6818)

### Dependencies

- #6837 — Runtime RPC dispatch layer and Unix socket transport (server side)
- #6818 — `--ephemeral` daemon mode (auto-spawn)
- #6817 — Session-scoped runtime overrides (configure method)
