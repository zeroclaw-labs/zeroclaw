# [Feature]: File/attachment upload protocol

File/attachment upload protocol for the RPC transport. Allows clients (TUI, web dashboard) to send binary file content to the daemon — for example, uploading files to the agent's workspace, attaching images to prompts, or streaming large tool outputs.

The current NDJSON framing is text-only. Binary payloads need either base64 encoding within JSON (simple, ~33% overhead) or a binary frame extension to the transport protocol (efficient, more complex).

### Design considerations

- Base64 in JSON is simplest and works with the existing NDJSON transport
- A binary frame type (length-prefixed binary interleaved with NDJSON) would be more efficient for large files
- The gateway already handles file uploads via multipart HTTP — this is the RPC equivalent
- Should work across Unix socket, WSS, and (future) vsock transports
