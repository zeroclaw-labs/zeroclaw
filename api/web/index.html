collect_property_descriptions 🔒
Merge every properties.<field>.description found in a schema (including its oneOf / anyOf variants, so serde-tagged enums like SopTrigger surface each variant’s field docs) into fields.
collect_string_enum_members 🔒
Collect the string variants of a schema whether it uses top-level enum, oneOf/anyOf const strings, or oneOf/anyOf enum string arrays. Object variants (e.g. StepFailure::Goto) contribute nothing. Preserves spec order and drops duplicates so pickers render one row per variant.

render_descriptions 🔒
Extract { SchemaName: { field: description } } from the OpenAPI spec so the frontend renders tooltips from the Rust /// docs at runtime. TypeScript @description JSDoc is erased at build time and unreadable at runtime; this projects the same spec into a real data module. The Rust doc comments remain the single source of truth, nothing is retyped on the frontend.
render_enum_values 🔒
Extract { SchemaName: [variant, ...] } for every schema whose top-level shape is a string enum, so option pickers walk the spec instead of retyping variant lists. The Rust enums remain the single source of truth.
