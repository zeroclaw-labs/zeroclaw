Derive Macro TriggerFields 

#[derive(TriggerFields)]
{
    // Attributes available to this derive:
    #[trigger]
}
Expand description

Derive field_specs(&self) -> Vec<TriggerField> for a trigger enum, so the authoring registry never hand-lists a source’s fields. Field kind is inferred by convention from each variant’s fields:

  • a field named condition becomes an expression field;
  • a Vec<T> whose element type ends in Kind becomes a multi-select options field populated from T::iter();
  • any other Vec<_> becomes a plain list field;
  • everything else becomes a text field.

A variant marked #[trigger(config_derived)] yields no fields: its options depend on live config and the registry supplies them separately. Add a variant and its field spec is generated automatically - there is no second place to edit.
