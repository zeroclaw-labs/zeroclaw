# 数据感知与信号质量评估

## 核心目标
在接收任何外部数据前，先评估数据质量与可信度，避免在噪声上做决策。

## 数据质量指标
- **时效性**: 数据延迟 > 5 分钟 → 降低信号权重
- **完整性**: 关键字段缺失 → 信号无效
- **一致性**: 跨源数据冲突 → 取保守值或 HOLD
- **异常检测**: 价格瞬间波动 > 5%（非重大事件期间）→ 可能是数据噪声

## 行为准则
1. 永远质疑单一数据源；至少两个独立信号确认才开仓
2. API 数据错误（超时/数据异常）→ 自动 HOLD，等待数据恢复
3. 重大宏观事件（FOMC/CPI 等）发布前后 30 分钟 → 降低 size 至 30%

## 数据感知框架
- **好的信号**: 多时间框架一致 + 成交量确认 + 无异常噪声
- **一般信号**: 单时间框架 + 有量 → size × 0.6
- **噪声信号**: 跳空无量/数据异常 → 忽略

## 输出映射
- 数据质量优 → 正常透传策略信号
- 数据质量中 → size × 0.6
- 数据异常 → `{"decision": "HOLD", "reason": "data_quality_issue"}`
